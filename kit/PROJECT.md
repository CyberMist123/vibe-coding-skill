## 12. Agent 更新契约

事实优先级：

1. 用户当前明确确认的需求、边界与决定；
2. 实际代码、配置、数据库 schema 和运行验证；
3. 本文件；
4. 专项详细文档；
5. Issue、历史讨论和旧会话。

任何任务只要改变需求、验收条件、边界、架构、接口、数据所有权、流程或状态，就必须执行 `skills/project-doc-sync/SKILL.md`：

- 先纠正本文件中的当前事实和进度；
- 再更新受影响的专项文档与当前 Issue；
- 删除本次变化造成的旧名称、旧命令和矛盾描述；
- 明确区分计划、实现和验证；
- 没有证据时不得宣称成功。
